# Models

Continuum AI provides access to a diverse catalog of models optimized for different tasks. Each model is tuned for specific workloads -- from fast, cost-efficient chat to deep reasoning, high-fidelity image generation, and real-time audio processing.

<Info>
  All models are accessed through the same base URL and authentication method. Specify the model name in your request body to route to the right model.
</Info>

## List models

Retrieve the list of all models available to your API key.

### GET /v1/models

```bash
curl https://api.continuumai.technology/v1/models \
  -H "Authorization: Bearer sk-proj-your-api-key"
```

```json Response
{
  "data": [
    {
      "id": "continuum-ultra",
      "object": "model",
      "created": 1709251200,
      "owned_by": "continuum-ai",
      "capabilities": ["chat", "function_calling", "vision"],
      "context_window": 200000,
      "max_output_tokens": 16384
    },
    {
      "id": "continuum-turbo",
      "object": "model",
      "created": 1709251200,
      "owned_by": "continuum-ai",
      "capabilities": ["chat", "function_calling"],
      "context_window": 128000,
      "max_output_tokens": 8192
    }
  ],
  "status": 200
}
```

### GET /v1/models/\{model\}

Retrieve details for a specific model.

```bash
curl https://api.continuumai.technology/v1/models/continuum-ultra \
  -H "Authorization: Bearer sk-proj-your-api-key"
```

```json Response
{
  "data": {
    "id": "continuum-ultra",
    "object": "model",
    "created": 1709251200,
    "owned_by": "continuum-ai",
    "capabilities": ["chat", "function_calling", "vision", "structured_output"],
    "context_window": 200000,
    "max_output_tokens": 16384,
    "training_cutoff": "2025-12-01",
    "pricing": {
      "input_per_1m_tokens": 10.00,
      "output_per_1m_tokens": 30.00
    }
  },
  "status": 200
}
```

  <Tab title="Chat">
    Chat models generate conversational responses, support function calling, structured output, and multi-turn dialogue.

    | Model | Context window | Max output | Capabilities | Description |
    |-------|---------------:|-----------:|--------------|-------------|
    | `continuum-ultra` | 200K | 16,384 | Chat, Vision, Function calling, Structured output | Flagship model. Best overall quality for complex tasks, analysis, and creative work. |
    | `continuum-ultra-mini` | 200K | 16,384 | Chat, Vision, Function calling, Structured output | Compact version of Ultra. Near-flagship quality at significantly lower cost. |
    | `continuum-turbo` | 128K | 8,192 | Chat, Function calling, Structured output | High-speed model optimized for low latency. Ideal for real-time applications. |
    | `continuum-pro` | 128K | 8,192 | Chat, Function calling, Structured output | Balanced performance and cost. Great default choice for most production workloads. |
    | `continuum-lite` | 32K | 4,096 | Chat, Function calling | Lightweight and fast. Best for simple tasks, classification, and high-volume processing. |

## Model versioning

Continuum AI models are versioned with dated snapshots. When you specify a model name like `continuum-ultra`, you are automatically routed to the latest stable version.

To pin a specific version, append the date suffix:

```
continuum-ultra-2026-03-15
```

<Note>
  Model versions are supported for at least 6 months after a newer version is released. You will receive deprecation notices via email and the API response headers at least 60 days before a version is retired.
</Note>

## Context window tips

<AccordionGroup>
  <Accordion title="What counts toward the context window?" icon="ruler">
    The context window includes everything in your request: the system prompt, all messages in the conversation history, any tool definitions, and the model's response. Plan your prompts accordingly.
  </Accordion>
  <Accordion title="What happens if I exceed the context window?" icon="triangle-exclamation">
    The API returns a `400 Bad Request` error with code `INVALID_REQUEST_BODY` and a message indicating the input exceeds the model's maximum context length. Truncate your input or switch to a model with a larger context window.
  </Accordion>
  <Accordion title="How do I estimate token count?" icon="calculator">
    As a rough guide, 1 token is approximately 4 characters or 0.75 words in English. Use the tokenizer endpoint or client library utilities to get exact counts before sending requests.
  </Accordion>
</AccordionGroup>
