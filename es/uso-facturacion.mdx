## Uso y facturacion

La API de Continuum AI te permite consultar el consumo detallado de tokens, costos y limites de uso de tu organizacion. Usa estos endpoints para monitorear gastos, establecer alertas y optimizar el uso de tus recursos.

<Info>
  El uso se rastrea por proyecto y por modelo, lo que te permite identificar exactamente donde se consumen tus recursos y optimizar costos de forma granular.
</Info>

## Endpoints

### Obtener resumen de uso

**GET /v1/usage**

Obtiene un resumen del consumo de tu organizacion en un periodo de tiempo.

<ParamField query="start_date" type="string" required>
  Fecha de inicio en formato `YYYY-MM-DD`. Ejemplo: `2026-03-01`.
</ParamField>

<ParamField query="end_date" type="string" required>
  Fecha de fin en formato `YYYY-MM-DD`. Ejemplo: `2026-03-22`.
</ParamField>

<ParamField query="project_id" type="string">
  Filtrar por proyecto especifico.
</ParamField>

<ParamField query="group_by" type="string">
  Agrupar resultados por: `model`, `project`, `day`.
</ParamField>

<CodeGroup>

```bash curl
curl "https://api.continuum-ai.com/v1/usage?start_date=2026-03-01&end_date=2026-03-22&group_by=model" \
  -H "Authorization: Bearer sk-proj-tu-api-key"
```

```python Python
from openai import OpenAI
import requests

client = OpenAI(
    api_key="sk-proj-tu-api-key",
    base_url="https://api.continuum-ai.com/v1"
)

# Consulta directa al endpoint de uso
headers = {"Authorization": "Bearer sk-proj-tu-api-key"}
params = {
    "start_date": "2026-03-01",
    "end_date": "2026-03-22",
    "group_by": "model"
}

response = requests.get(
    "https://api.continuum-ai.com/v1/usage",
    headers=headers,
    params=params
)

usage = response.json()
for entry in usage["data"]:
    print(f"Modelo: {entry['model']}")
    print(f"  Solicitudes: {entry['total_requests']}")
    print(f"  Tokens entrada: {entry['input_tokens']:,}")
    print(f"  Tokens salida: {entry['output_tokens']:,}")
    print(f"  Costo: ${entry['cost']:.4f}")
```

```javascript Node.js

```

### Obtener limites de uso

**GET /v1/usage/limits**

Consulta los limites de uso activos para tu organizacion y el consumo actual respecto a esos limites.

<CodeGroup>

```bash curl
curl https://api.continuum-ai.com/v1/usage/limits \
  -H "Authorization: Bearer sk-proj-tu-api-key"
```

```python Python
headers = {"Authorization": "Bearer sk-proj-tu-api-key"}
response = requests.get(
    "https://api.continuum-ai.com/v1/usage/limits",
    headers=headers
)

limits = response.json()
print(f"Plan: {limits['data']['plan']}")
print(f"Limite mensual: ${limits['data']['monthly_limit']}")
print(f"Consumo actual: ${limits['data']['current_usage']}")
print(f"Porcentaje usado: {limits['data']['usage_percentage']:.1f}%")
```

```javascript Node.js
const response = await fetch("https://api.continuum-ai.com/v1/usage/limits", {
  headers: {
    Authorization: "Bearer sk-proj-tu-api-key",
  },
});

const limits = await response.json();
console.log(`Plan: ${limits.data.plan}`);
console.log(`Limite mensual: $${limits.data.monthly_limit}`);
console.log(`Consumo actual: $${limits.data.current_usage}`);
console.log(`Porcentaje usado: ${limits.data.usage_percentage.toFixed(1)}%`);
```

### Obtener desglose de costos

**GET /v1/usage/costs**

Obtiene un desglose detallado de costos por modelo, incluyendo precios por millon de tokens.

<ParamField query="start_date" type="string" required>
  Fecha de inicio.
</ParamField>

<ParamField query="end_date" type="string" required>
  Fecha de fin.
</ParamField>

<CodeGroup>

```bash curl
curl "https://api.continuum-ai.com/v1/usage/costs?start_date=2026-03-01&end_date=2026-03-22" \
  -H "Authorization: Bearer sk-proj-tu-api-key"
```

```python Python
headers = {"Authorization": "Bearer sk-proj-tu-api-key"}
params = {"start_date": "2026-03-01", "end_date": "2026-03-22"}

response = requests.get(
    "https://api.continuum-ai.com/v1/usage/costs",
    headers=headers,
    params=params
)

costs = response.json()
for model in costs["data"]["models"]:
    print(f"\n{model['model']}:")
    print(f"  Precio entrada: ${model['input_price_per_1m']} / 1M tokens")
    print(f"  Precio salida: ${model['output_price_per_1m']} / 1M tokens")
    print(f"  Costo total: ${model['total_cost']:.4f}")
```


## Alertas de uso

<Tip>
  Configura alertas de uso desde el panel de administracion para recibir notificaciones cuando alcances ciertos umbrales de consumo (por ejemplo, 50%, 80% y 100% de tu limite mensual). Esto te ayuda a evitar interrupciones inesperadas del servicio.
</Tip>

## Optimizacion de costos

<AccordionGroup>
  <Accordion title="Elige el modelo adecuado">
    No uses `continuum-ultra` para tareas simples. Para clasificacion basica, resumen corto o respuestas de formato fijo, `continuum-lite` o `continuum-turbo` son significativamente mas economicos y suficientemente capaces.
  </Accordion>
  <Accordion title="Usa procesamiento por lotes">
    Si no necesitas respuestas inmediatas, la [API de Lotes](/es/lotes) ofrece un 50% de descuento. Ideal para procesamiento nocturno o tareas de ETL.
  </Accordion>
  <Accordion title="Optimiza tus prompts">
    Prompts mas cortos y directos consumen menos tokens de entrada. Elimina instrucciones redundantes y usa el campo `system` de forma eficiente.
  </Accordion>
  <Accordion title="Establece max_tokens">
    Siempre configura `max_tokens` para evitar respuestas innecesariamente largas. Esto limita el costo maximo por solicitud.
  </Accordion>
  <Accordion title="Cachea respuestas frecuentes">
    Si multiples usuarios hacen las mismas preguntas, implementa una capa de cache para evitar solicitudes repetidas a la API.
  </Accordion>
  <Accordion title="Reduce dimensiones de embeddings">
    Usa el parametro `dimensions` con modelos `embed-3` para generar vectores mas pequenos. 256 o 512 dimensiones suelen ser suficientes para la mayoria de casos.
  </Accordion>
</AccordionGroup>
