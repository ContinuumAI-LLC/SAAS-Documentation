## Conversaciones persistentes

Los Threads (hilos de conversacion) permiten gestionar conversaciones con estado en el servidor. En lugar de enviar todo el historial de mensajes en cada solicitud, creas un thread y el servidor se encarga de mantener el contexto.

<Info>
  Los Threads son ideales para construir asistentes de chat, agentes conversacionales y cualquier aplicacion que requiera conversaciones multiturno con persistencia.
</Info>

## Flujo de trabajo

<Steps>
  <Step title="Crear un Thread">
    Crea un nuevo hilo de conversacion. Opcionalmente puedes incluir mensajes iniciales.
  </Step>
  <Step title="Agregar mensajes">
    Anade mensajes del usuario al thread existente.
  </Step>
  <Step title="Crear un Run">
    Ejecuta el modelo sobre el thread para generar una respuesta del asistente.
  </Step>
  <Step title="Leer la respuesta">
    Recupera los mensajes del thread para obtener la respuesta generada.
  </Step>
  <Step title="Continuar la conversacion">
    Repite los pasos 2-4 para mantener la conversacion de forma indefinida.
  </Step>
</Steps>

## Endpoints

### Crear un Thread

**POST /v1/threads**

Crea un nuevo hilo de conversacion vacio o con mensajes iniciales.

<ParamField body="messages" type="array">
  Lista opcional de mensajes iniciales para el thread.
</ParamField>

<ParamField body="metadata" type="object">
  Metadatos personalizados asociados al thread. Maximo 16 pares clave-valor.
</ParamField>

<CodeGroup>

```bash curl
curl https://api.continuumai.technology/v1/threads \
  -H "Authorization: Bearer sk-proj-tu-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {
        "role": "user",
        "content": "Hola, necesito ayuda con mi proyecto de Python."
      }
    ],
    "metadata": {
      "user_id": "usr_123",
      "topic": "python-help"
    }
  }'
```

```python Python
import requests

headers = {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
}

thread = requests.post(
    "https://api.continuumai.technology/v1/threads",
    headers=headers,
    json={
        "messages": [
            {
                "role": "user",
                "content": "Hola, necesito ayuda con mi proyecto de Python."
            }
        ],
        "metadata": {
            "user_id": "usr_123",
            "topic": "python-help"
        },
    },
).json()

print(f"Thread creado: {thread['id']}")
```

```javascript Node.js
const response = await fetch("https://api.continuumai.technology/v1/threads", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    messages: [
      {
        role: "user",
        content: "Hola, necesito ayuda con mi proyecto de Python.",
      },
    ],
    metadata: {
      user_id: "usr_123",
      topic: "python-help",
    },
  }),
});

const thread = await response.json();
console.log(`Thread creado: ${thread.id}`);
```

</CodeGroup>

**Respuesta:**

```json
{
  "id": "thread_abc123",
  "object": "thread",
  "created_at": 1711152000,
  "metadata": {
    "user_id": "usr_123",
    "topic": "python-help"
  }
}
```

### Modificar un Thread

**POST /v1/threads/{thread_id}**

Actualiza los metadatos de un thread.

<ParamField path="thread_id" type="string" required>
  Identificador del thread a modificar.
</ParamField>

<ParamField body="metadata" type="object">

```bash curl
    "metadata": {

```

### Eliminar un Thread

**DELETE /v1/threads/{thread_id}**

Elimina un thread y todos sus mensajes de forma permanente.

<ParamField path="thread_id" type="string" required>
  Identificador del thread a eliminar.
</ParamField>

```bash curl
curl -X DELETE https://api.continuumai.technology/v1/threads/thread_abc123 \
  -H "Authorization: Bearer sk-proj-tu-api-key"
```

<Warning>
  La eliminacion es permanente e irreversible. Todos los mensajes y runs asociados al thread seran eliminados.
</Warning>

### Crear un mensaje

**POST /v1/threads/{thread_id}/messages**

Anade un nuevo mensaje a un thread existente.

<ParamField path="thread_id" type="string" required>
  Identificador del thread.
</ParamField>

<ParamField body="role" type="string" required>
  Rol del mensaje. Opciones: `user`, `assistant`.
</ParamField>

<ParamField body="content" type="string | array" required>
  Contenido del mensaje. Puede ser texto simple o contenido multimodal.
</ParamField>

<ParamField body="metadata" type="object">
  Metadatos opcionales del mensaje.
</ParamField>

<CodeGroup>

```bash curl

  -H "Authorization: Bearer sk-proj-tu-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "user",
    "content": "Como puedo leer un archivo CSV con pandas?"

```

```python Python
import requests
```

### Listar mensajes

**GET /v1/threads/{thread_id}/messages**

Obtiene todos los mensajes de un thread, ordenados del mas reciente al mas antiguo.

<ParamField path="thread_id" type="string" required>
  Identificador del thread.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Numero maximo de mensajes a devolver. Entre 1 y 100.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Orden de los resultados: `asc` (antiguo primero) o `desc` (reciente primero).
</ParamField>

```python Python
import requests

messages = requests.get(
    "https://api.continuumai.technology/v1/threads/thread_abc123/messages?order=asc",
    headers={"Authorization": "Bearer sk-proj-tu-api-key"},
).json()

for msg in messages["data"]:
    print(f"[{msg['role']}] {msg['content'][0]['text']['value']}")
```

### Crear un Run

**POST /v1/threads/{thread_id}/runs**

Ejecuta el modelo sobre el thread completo para generar una respuesta. El modelo procesa todos los mensajes del thread y genera una respuesta del asistente.

<ParamField path="thread_id" type="string" required>
  Identificador del thread.
</ParamField>

<ParamField body="model" type="string" required>
  Modelo a utilizar para la generacion.
</ParamField>

<ParamField body="instructions" type="string">
  Instrucciones del sistema para esta ejecucion.
</ParamField>

<ParamField body="tools" type="array">
  Herramientas disponibles para el modelo durante la ejecucion.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  Habilita streaming de eventos durante la ejecucion.
</ParamField>

<ParamField body="max_completion_tokens" type="integer">
  Numero maximo de tokens en la respuesta.
</ParamField>

<ParamField body="temperature" type="number">
  Controla la aleatoriedad de la respuesta.
</ParamField>

<CodeGroup>

```bash curl
curl https://api.continuumai.technology/v1/threads/thread_abc123/runs \
  -H "Authorization: Bearer sk-proj-tu-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "continuum-ultra",
    "instructions": "Eres un experto en Python. Responde con ejemplos de codigo claros y explicaciones concisas.",
    "temperature": 0.7
  }'
```

```python Python
import requests

run = requests.post(
    "https://api.continuumai.technology/v1/threads/thread_abc123/runs",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-ultra",
        "instructions": "Eres un experto en Python. Responde con ejemplos de codigo claros y explicaciones concisas.",
        "temperature": 0.7,
    },
).json()

print(f"Run creado: {run['id']} - Estado: {run['status']}")
```

```javascript Node.js
const response = await fetch("https://api.continuumai.technology/v1/threads/thread_abc123/runs", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "continuum-ultra",
    instructions:
      "Eres un experto en Python. Responde con ejemplos de codigo claros y explicaciones concisas.",
    temperature: 0.7,
  }),
});

```

</CodeGroup>

**Respuesta:**

```json
{
  "id": "run_xyz789",
  "object": "thread.run",
  "created_at": 1711152000,
  "thread_id": "thread_abc123",
  "model": "continuum-ultra",
  "status": "queued",
  "instructions": "Eres un experto en Python...",
  "usage": null
}
```

### Estados de un Run

| Estado | Descripcion |
|--------|-------------|
| `queued` | En cola, esperando a ser procesado. |
| `in_progress` | El modelo esta generando la respuesta. |
| `requires_action` | El modelo necesita que ejecutes una herramienta. |
| `completed` | La generacion ha finalizado correctamente. |
| `failed` | Ocurrio un error durante la generacion. |
| `cancelled` | El run fue cancelado por el usuario. |
| `expired` | El run expiro sin completarse. |

### Run con streaming

Para recibir la respuesta en tiempo real, usa el parametro `stream: true`:

```python Python
import requests
import json

response = requests.post(
    "https://api.continuumai.technology/v1/threads/thread_abc123/runs",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-turbo",
        "instructions": "Eres un asistente amable y conciso.",
        "stream": True,
    },
    stream=True,
)

for line in response.iter_lines():
    if line:
        text = line.decode("utf-8")
        if text.startswith("data: ") and text != "data: [DONE]":
            event = json.loads(text[6:])
            if event.get("event") == "thread.message.delta":
                content = event["data"]["delta"]["content"][0]
                if content["type"] == "text":
                    print(content["text"]["value"], end="", flush=True)
```

```javascript Node.js
const response = await fetch("https://api.continuumai.technology/v1/threads/thread_abc123/runs", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "continuum-turbo",
    instructions: "Eres un asistente amable y conciso.",
    stream: true,
  }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  for (const line of text.split("\n")) {
    if (line.startsWith("data: ") && line !== "data: [DONE]") {
      const event = JSON.parse(line.slice(6));
      if (event.event === "thread.message.delta") {
        const content = event.data.delta.content[0];
        if (content.type === "text") {
          process.stdout.write(content.text.value);
        }
      }
    }
  }
}
```

### Obtener estado de un Run

**GET /v1/threads/{thread_id}/runs/{run_id}**

Consulta el estado actual de un run en ejecucion.

```python Python
import requests

run = requests.get(
    "https://api.continuumai.technology/v1/threads/thread_abc123/runs/run_xyz789",
    headers={"Authorization": "Bearer sk-proj-tu-api-key"},
).json()

print(f"Estado: {run['status']}")
if run["status"] == "completed":
    print(f"Tokens usados: {run['usage']['total_tokens']}")
```

## Ejemplo completo: Asistente de chat

```python Python
import requests
import json

headers = {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
}

# 1. Crear el thread
thread = requests.post(
    "https://api.continuumai.technology/v1/threads",
    headers=headers,
    json={},
).json()

thread_id = thread["id"]

# 2. Bucle de conversacion
while True:
    user_input = input("\nTu: ")
    if user_input.lower() in ["salir", "exit", "quit"]:
        break

    # 3. Agregar mensaje del usuario
    requests.post(
        f"https://api.continuumai.technology/v1/threads/{thread_id}/messages",
        headers=headers,
        json={"role": "user", "content": user_input},
    )

    # 4. Ejecutar el modelo con streaming
    print("\nAsistente: ", end="")
    response = requests.post(
        f"https://api.continuumai.technology/v1/threads/{thread_id}/runs",
        headers=headers,
        json={
            "model": "continuum-turbo",
            "instructions": "Eres un asistente amable. Responde en espanol de forma clara y concisa.",
            "stream": True,
        },
        stream=True,
    )

    for line in response.iter_lines():
        if line:
            text = line.decode("utf-8")
            if text.startswith("data: ") and text != "data: [DONE]":
                event = json.loads(text[6:])
                if event.get("event") == "thread.message.delta":
                    content = event["data"]["delta"]["content"][0]
                    if content["type"] == "text":
                        print(content["text"]["value"], end="", flush=True)

print("\n\nConversacion finalizada.")
```

<Tip>
  Los Threads se almacenan en el servidor de forma indefinida. Puedes retomar cualquier conversacion en cualquier momento usando el `thread_id` original.
</Tip>
