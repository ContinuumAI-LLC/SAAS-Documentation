## Generacion de imagenes

Los modelos de vision de Continuum AI te permiten generar imagenes a partir de descripciones textuales, editar imagenes existentes y crear variaciones.

<Info>
  Todos los endpoints de imagenes devuelven URLs temporales (validas por 1 hora) o datos en formato base64, segun el parametro `response_format`.
</Info>

## Modelos disponibles

| Modelo | Resolucion max. | Descripcion |
|--------|-----------------|-------------|
| `continuum-vision-hd` | 2048x2048 | Alta definicion con maximo nivel de detalle y realismo |
| `continuum-vision` | 1024x1024 | Generacion rapida para uso general y prototipos |

<Tabs>
  <Tab title="Generar imagen">
    ## POST /v1/images/generations

    Genera una o varias imagenes a partir de una descripcion textual (prompt).

    ### Parametros

    <ParamField body="model" type="string" required>
      Modelo de generacion. Opciones: `continuum-vision-hd`, `continuum-vision`.
    </ParamField>

    <ParamField body="prompt" type="string" required>
      Descripcion textual de la imagen a generar. Maximo 4,000 caracteres. Se mas descriptivo para obtener mejores resultados.
    </ParamField>

    <ParamField body="n" type="integer" default="1">
      Numero de imagenes a generar. Entre 1 y 10.
    </ParamField>

    <ParamField body="size" type="string" default="1024x1024">
      Tamano de la imagen. Opciones: `256x256`, `512x512`, `1024x1024`, `1024x1792`, `1792x1024`, `2048x2048` (solo vision-hd).
    </ParamField>

    <ParamField body="quality" type="string" default="standard">
      Calidad de la imagen. Opciones: `standard`, `hd`. La calidad `hd` genera imagenes con mayor detalle y coherencia.
    </ParamField>

    <ParamField body="style" type="string" default="vivid">
      Estilo visual. Opciones: `vivid` (colores vibrantes e hiperrealistas) o `natural` (tonos mas sutiles y naturales).
    </ParamField>

    <ParamField body="response_format" type="string" default="url">
      Formato de la respuesta: `url` (enlace temporal) o `b64_json` (datos base64).
    </ParamField>

    ### Ejemplos

    <CodeGroup>

    ```bash curl
    curl https://api.continuumai.technology/v1/images/generations \
      -H "Authorization: Bearer sk-proj-tu-api-key" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "continuum-vision-hd",
        "prompt": "Un atardecer sobre el mar Mediterraneo visto desde un acantilado, estilo fotorrealista, iluminacion dorada",
        "n": 1,
        "size": "1024x1024",
        "quality": "hd"
      }'
    ```

    ```python Python
    import requests

    response = requests.post(
        "https://api.continuumai.technology/v1/images/generations",
        headers={
            "Authorization": "Bearer sk-proj-tu-api-key",
            "Content-Type": "application/json",
        },
        json={
            "model": "continuum-vision-hd",
            "prompt": "Un atardecer sobre el mar Mediterraneo visto desde un acantilado, estilo fotorrealista, iluminacion dorada",
            "n": 1,
            "size": "1024x1024",
            "quality": "hd",
        },
    )

    data = response.json()
    print(data["data"][0]["url"])
    ```

    ```javascript Node.js
    const response = await fetch("https://api.continuumai.technology/v1/images/generations", {
      method: "POST",
      headers: {
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        model: "continuum-vision-hd",
        prompt: "Un atardecer sobre el mar Mediterraneo visto desde un acantilado, estilo fotorrealista, iluminacion dorada",
        n: 1,
        size: "1024x1024",
        quality: "hd",
      }),
    });

    const data = await response.json();
    console.log(data.data[0].url);
    ```

    </CodeGroup>

    **Respuesta:**

    ```json
    {
      "created": 1711152000,
      "data": [
        {
          "url": "https://api.continuumai.technology/files/img-abc123.png",
          "revised_prompt": "Un hermoso atardecer sobre el mar Mediterraneo visto desde lo alto de un acantilado rocoso..."
        }
      ]
    }
    ```

    <ResponseField name="url" type="string">
      URL temporal de la imagen generada. Valida durante 1 hora.
    </ResponseField>

    <ResponseField name="revised_prompt" type="string">
      Version revisada y expandida del prompt original utilizada internamente para la generacion.
    </ResponseField>

    <Tip>
      Los prompts en ingles tienden a producir mejores resultados. Si escribes en espanol, el modelo traducira internamente tu descripcion.
    </Tip>
  </Tab>

  <Tab title="Editar imagen">
    ## POST /v1/images/edits

    Edita una imagen existente segun instrucciones textuales. Requiere una imagen base y opcionalmente una mascara que indique la zona a modificar.

    ### Parametros

    <ParamField body="image" type="file" required>
      Imagen original en formato PNG. Debe ser cuadrada y menor a 4 MB.
    </ParamField>

    <ParamField body="prompt" type="string" required>
      Descripcion de la edicion a realizar.
    </ParamField>

    <ParamField body="mask" type="file">
      Imagen PNG con canal alfa que indica las areas a editar. Las zonas transparentes seran reemplazadas.
    </ParamField>

    <ParamField body="model" type="string" default="continuum-vision">
      Modelo a utilizar para la edicion.
    </ParamField>

    <ParamField body="n" type="integer" default="1">
      Numero de variaciones editadas a generar.
    </ParamField>

    <ParamField body="size" type="string" default="1024x1024">
      Tamano de la imagen resultante.
    </ParamField>

    ### Ejemplo

    <CodeGroup>

    ```bash curl
    curl https://api.continuumai.technology/v1/images/edits \
      -H "Authorization: Bearer sk-proj-tu-api-key" \
      -F image="@imagen_original.png" \
      -F mask="@mascara.png" \
      -F prompt="Anade un gato naranja sentado en la silla" \
      -F model="continuum-vision" \
      -F n=1 \
      -F size="1024x1024"
    ```

    ```python Python
    import requests

    response = requests.post(
        "https://api.continuumai.technology/v1/images/edits",
        headers={"Authorization": "Bearer sk-proj-tu-api-key"},
        files={
            "image": open("imagen_original.png", "rb"),
            "mask": open("mascara.png", "rb"),
        },
        data={
            "prompt": "Anade un gato naranja sentado en la silla",
            "model": "continuum-vision",
            "n": "1",
            "size": "1024x1024",
        },
    )

    data = response.json()
    print(data["data"][0]["url"])
    ```

    ```javascript Node.js
    import fs from "fs";

    const formData = new FormData();
    formData.append("image", new Blob([fs.readFileSync("imagen_original.png")]));
    formData.append("mask", new Blob([fs.readFileSync("mascara.png")]));
    formData.append("prompt", "Anade un gato naranja sentado en la silla");
    formData.append("model", "continuum-vision");
    formData.append("n", "1");
    formData.append("size", "1024x1024");

    const response = await fetch("https://api.continuumai.technology/v1/images/edits", {
      method: "POST",
      headers: { "Authorization": "Bearer sk-proj-tu-api-key" },
      body: formData,
    });

    const data = await response.json();
    console.log(data.data[0].url);
    ```

    </CodeGroup>

    <Warning>
      La imagen y la mascara deben tener el mismo tamano y ser cuadradas. Formatos soportados: PNG con canal alfa para la mascara.
    </Warning>
  </Tab>

  <Tab title="Variaciones">
    ## POST /v1/images/variations

    Genera variaciones de una imagen existente, manteniendo el estilo y la composicion general pero con diferencias creativas.

    ### Parametros

    <ParamField body="image" type="file" required>
      Imagen base en formato PNG. Debe ser cuadrada y menor a 4 MB.
    </ParamField>

    <ParamField body="model" type="string" default="continuum-vision">
      Modelo a utilizar.
    </ParamField>

    <ParamField body="n" type="integer" default="1">
      Numero de variaciones a generar. Entre 1 y 10.
    </ParamField>

    <ParamField body="size" type="string" default="1024x1024">
      Tamano de las imagenes resultantes.
    </ParamField>

    ### Ejemplo

    <CodeGroup>

    ```bash curl
    curl https://api.continuumai.technology/v1/images/variations \
      -H "Authorization: Bearer sk-proj-tu-api-key" \
      -F image="@imagen_original.png" \
      -F model="continuum-vision" \
      -F n=3 \
      -F size="1024x1024"
    ```

    ```python Python
    import requests

    response = requests.post(
        "https://api.continuumai.technology/v1/images/variations",
        headers={"Authorization": "Bearer sk-proj-tu-api-key"},
        files={"image": open("imagen_original.png", "rb")},
        data={
            "model": "continuum-vision",
            "n": "3",
            "size": "1024x1024",
        },
    )

    data = response.json()
    for i, img in enumerate(data["data"]):
        print(f"Variacion {i + 1}: {img['url']}")
    ```

    ```javascript Node.js
    import fs from "fs";

    const formData = new FormData();
    formData.append("image", new Blob([fs.readFileSync("imagen_original.png")]));
    formData.append("model", "continuum-vision");
    formData.append("n", "3");
    formData.append("size", "1024x1024");

    const response = await fetch("https://api.continuumai.technology/v1/images/variations", {
      method: "POST",
      headers: { "Authorization": "Bearer sk-proj-tu-api-key" },
      body: formData,
    });

    const data = await response.json();
    data.data.forEach((img, i) => {
      console.log(`Variacion ${i + 1}: ${img.url}`);
    });
    ```

    </CodeGroup>

    <Tip>
      Las variaciones son utiles para explorar diferentes interpretaciones creativas de un concepto visual sin necesidad de escribir multiples prompts.
    </Tip>
  </Tab>

## Consejos para mejores resultados

<AccordionGroup>
  <Accordion title="Escribe prompts descriptivos">
    Incluye detalles sobre el sujeto, estilo artistico, iluminacion, composicion y ambiente. Por ejemplo, en lugar de "un perro", escribe "un golden retriever jugando en un parque verde al atardecer, fotografia profesional con bokeh suave".
  </Accordion>
  <Accordion title="Especifica el estilo visual">
    Indica claramente si deseas un resultado fotorrealista, ilustracion, pintura al oleo, pixel art, 3D render, etc.
  </Accordion>
  <Accordion title="Usa tamanios apropiados">
    Elige `1024x1792` para retratos verticales, `1792x1024` para paisajes horizontales, y `1024x1024` para imagenes cuadradas como avatares o iconos.
  </Accordion>
  <Accordion title="Calidad HD vs Standard">
    Usa `quality: "hd"` para contenido final de produccion. Para borradores, iteraciones rapidas o prototipos, `standard` es mas rapida y economica.
  </Accordion>
</AccordionGroup>
