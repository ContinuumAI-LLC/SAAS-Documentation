## POST /v1/responses

El endpoint de Responses es la forma moderna y recomendada de interactuar con los modelos de Continuum AI. Ofrece una interfaz unificada que combina chat, herramientas, vision y streaming en un solo endpoint simplificado.

<Tip>
  **Endpoint recomendado.** Si estas comenzando un proyecto nuevo, te sugerimos usar `/v1/responses` en lugar de `/v1/chat/completions`. Ofrece una API mas limpia, soporte nativo para herramientas integradas y mejor manejo de conversaciones multiturno.
</Tip>

## Parametros de la solicitud

<ParamField body="model" type="string" required>
  Identificador del modelo. Ejemplo: `continuum-ultra`, `continuum-turbo`.
</ParamField>

<ParamField body="input" type="string | array" required>
  El contenido de entrada. Puede ser un string simple o un array de mensajes con contenido multimodal.
</ParamField>

<ParamField body="instructions" type="string">
  Instrucciones del sistema que definen el comportamiento del modelo. Equivalente al mensaje `system` en chat completions.
</ParamField>

<ParamField body="max_output_tokens" type="integer">
  Numero maximo de tokens en la respuesta generada.
</ParamField>

<ParamField body="temperature" type="number" default="1.0">
  Controla la aleatoriedad. Valores entre `0` y `2`.
</ParamField>

<ParamField body="top_p" type="number" default="1.0">
  Muestreo por nucleo. Valores entre `0` y `1`.
</ParamField>

<ParamField body="tools" type="array">
  Lista de herramientas disponibles para el modelo. Soporta funciones personalizadas y herramientas integradas.
</ParamField>

<ParamField body="tool_choice" type="string | object" default="auto">
  Controla la seleccion de herramientas: `auto`, `none`, `required`.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  Habilita streaming de la respuesta via Server-Sent Events.
</ParamField>

<ParamField body="metadata" type="object">
  Metadatos personalizados asociados a la solicitud. Se devuelven en la respuesta sin modificaciones.
</ParamField>

<ParamField body="store" type="boolean" default="true">
  Si es `true`, la solicitud y respuesta se almacenan para su uso posterior en conversaciones.
</ParamField>

<ParamField body="previous_response_id" type="string">
  ID de una respuesta anterior para continuar una conversacion multiturno sin reenviar el historial completo.
</ParamField>

## Solicitud basica

<CodeGroup>

```bash curl
curl https://api.continuumai.technology/v1/responses \
  -H "Authorization: Bearer sk-proj-tu-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "continuum-ultra",
    "input": "Que es el aprendizaje profundo?"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-ultra",
        "input": "Que es el aprendizaje profundo?",
    },
)

data = response.json()
print(data["output"][0]["content"][0]["text"])
```

```javascript Node.js
const response = await fetch("https://api.continuumai.technology/v1/responses", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "continuum-ultra",
    input: "Que es el aprendizaje profundo?",
  }),
});

const data = await response.json();
console.log(data.output[0].content[0].text);
```

## Conversaciones multiturno

Usa `previous_response_id` para continuar una conversacion sin reenviar todo el historial. El servidor mantiene el contexto automaticamente.

```python Python
import requests

headers = {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
}

# Primera solicitud
response1 = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers=headers,
    json={
        "model": "continuum-ultra",
        "input": "Mi nombre es Rafael y soy desarrollador de software.",
    },
).json()

# Segunda solicitud - continua la conversacion
response2 = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers=headers,
    json={
        "model": "continuum-ultra",
        "input": "Como me llamo y a que me dedico?",
        "previous_response_id": response1["id"],
    },
).json()

# El modelo recuerda el contexto anterior
print(response2["output"][0]["content"][0]["text"])
```

<Info>
  El parametro `previous_response_id` elimina la necesidad de enviar todo el historial de mensajes en cada solicitud, lo que reduce el consumo de tokens de entrada y simplifica la logica de tu aplicacion.
</Info>

## Streaming

<CodeGroup>

```python Python
import requests
import json

response = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-turbo",
        "input": "Escribe una guia rapida sobre Docker.",
        "stream": True,
    },
    stream=True,
)

for line in response.iter_lines():
    if line:
        text = line.decode("utf-8")
        if text.startswith("data: ") and text != "data: [DONE]":
            event = json.loads(text[6:])
            if event.get("type") == "response.output_text.delta":
                print(event["delta"], end="", flush=True)
```

```javascript Node.js
const response = await fetch("https://api.continuumai.technology/v1/responses", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-proj-tu-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "continuum-turbo",
    input: "Escribe una guia rapida sobre Docker.",
    stream: true,
  }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  for (const line of text.split("\n")) {
    if (line.startsWith("data: ") && line !== "data: [DONE]") {
      const event = JSON.parse(line.slice(6));
      if (event.type === "response.output_text.delta") {
        process.stdout.write(event.delta);
      }
    }
  }
}
```

</CodeGroup>

**Tipos de eventos en streaming:**

| Evento | Descripcion |
|--------|-------------|
| `response.created` | La respuesta se ha creado e iniciado el procesamiento. |
| `response.in_progress` | La generacion esta en curso. |
| `response.output_text.delta` | Fragmento parcial de texto generado. |
| `response.output_text.done` | La generacion de texto ha finalizado. |
| `response.completed` | La respuesta completa esta lista. |
| `response.failed` | Ocurrio un error durante la generacion. |

## Tool Calling con Responses

<CodeGroup>

```python Python
import requests

response = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-ultra",
        "input": "Busca el precio actual de las acciones de Apple.",
        "tools": [
            {
                "type": "function",
                "name": "get_stock_price",
                "description": "Obtiene el precio actual de una accion",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "symbol": {
                            "type": "string",
                            "description": "Simbolo bursatil (ej. AAPL, GOOGL)"
                        }
                    },
                    "required": ["symbol"]
                }
            }
        ],
    },
)

data = response.json()

# Verificar si el modelo quiere usar una herramienta
for item in data["output"]:
    if item["type"] == "function_call":
        print(f"Herramienta: {item['name']}")
        print(f"Argumentos: {item['arguments']}")
```

## Instrucciones del sistema

Usa el parametro `instructions` para definir el comportamiento del modelo de forma limpia y separada del input del usuario.

```python Python
import requests

response = requests.post(
    "https://api.continuumai.technology/v1/responses",
    headers={
        "Authorization": "Bearer sk-proj-tu-api-key",
        "Content-Type": "application/json",
    },
    json={
        "model": "continuum-ultra",
        "instructions": "Eres un experto en ciberseguridad. Responde de forma tecnica pero accesible. Incluye ejemplos practicos cuando sea relevante.",
        "input": "Como puedo proteger mi aplicacion web contra ataques XSS?",
    },
)

data = response.json()
print(data["output"][0]["content"][0]["text"])
```

<Note>
  La principal ventaja de `instructions` sobre un mensaje `system` en el array `input` es que las instrucciones se mantienen automaticamente en conversaciones multiturno cuando usas `previous_response_id`.
</Note>

## Comparacion con Chat Completions

| Caracteristica | Chat Completions | Responses |
|----------------|-----------------|-----------|
| Endpoint | `/v1/chat/completions` | `/v1/responses` |
| Entrada | Array de mensajes | String o array |
| Instrucciones sistema | Mensaje con `role: system` | Parametro `instructions` |
| Conversacion multiturno | Reenviar historial completo | `previous_response_id` |
| Herramientas integradas | No | Si |
| Streaming | SSE | SSE con eventos tipados |
| API REST directa | Si | Si |

<Tip>
  Para proyectos nuevos, recomendamos usar el endpoint de Responses. Para proyectos existentes que ya usan Chat Completions, ambos endpoints seguiran siendo soportados a largo plazo.
</Tip>
