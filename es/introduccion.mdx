## Construye con Continuum AI

La API de Continuum AI te brinda acceso a modelos de inteligencia artificial de ultima generacion para chat, embeddings, generacion de imagenes, audio y mucho mas. Integra capacidades de IA en tus aplicaciones con una API compatible con OpenAI, sin complicaciones.

<Note>
  **URL base de la API:** `https://api.continuumai.technology/v1`

  Todas las solicitudes deben incluir tu API key en el encabezado `Authorization`.
</Note>

## Formato de respuesta

Todas las respuestas de la API siguen un formato estandarizado con envolvente JSON:

<Tabs>
  <Tab title="Respuesta exitosa">
    ```json
    {
      "data": {
        "id": "chatcmpl-abc123",
        "object": "chat.completion",
        "choices": [...]
      },
      "status": 200
    }
    ```
  </Tab>
  <Tab title="Error">
    ```json
    {
      "error": {
        "message": "API key invalida.",
        "code": "INVALID_API_KEY"
      },
      "status": 401
    }
    ```
  </Tab>
  <Tab title="Paginada">
    ```json
    {
      "data": [...],
      "pagination": {
        "page": 1,
        "pageSize": 20,
        "total": 150,
        "totalPages": 8
      },
      "status": 200
    }
    ```
  </Tab>
</Tabs>

## Rate Limits

La API aplica limites de tasa para garantizar un servicio estable para todos los usuarios.

| Plan | Solicitudes/min | Tokens/min | Solicitudes/dia |
|------|-----------------|------------|-----------------|
| Gratuito | 60 | 40,000 | 1,000 |
| Pro | 600 | 400,000 | 50,000 |
| Enterprise | Personalizado | Personalizado | Personalizado |

<Warning>
  Si excedes los limites de tasa, recibiras un error `429 Too Many Requests`. Implementa logica de reintento con backoff exponencial para manejar estos casos correctamente.
</Warning>

## Compatibilidad con OpenAI

La API de Continuum AI es completamente compatible con el formato de OpenAI. Solo necesitas cambiar la URL base y tu API key:

```bash
curl -X POST https://api.continuumai.technology/v1/chat/completions \
  -H "Authorization: Bearer sk-proj-tu-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hola!"}]
  }'
```

<Tip>
  Todas las solicitudes se hacen directamente a la API via HTTP. No se requiere ningun SDK.
</Tip>
